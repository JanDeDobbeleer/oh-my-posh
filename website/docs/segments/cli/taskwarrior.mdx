## What

Display [Taskwarrior][taskwarrior] task counts for configurable filters and the active context.

## Sample Configuration

import Config from "@site/src/components/Config.js";

<Config
  data={{
    type: "taskwarrior",
    style: "powerline",
    powerline_symbol: "\uE0B0",
    foreground: "#193549",
    background: "#ffeb3b",
    template: " \uf4a0 {{ range $k, $v := .Tags }}{{ $k }}:{{ $v }} {{ end }}",
    options: {
      tags: {
        due: "+PENDING due.before:tomorrow",
        scheduled: "+PENDING scheduled.before:tomorrow",
        waiting: "+WAITING",
      },
      context_command: "_get rc.context",
    },
  }}
/>

## Options

| Name               | Type                  | Default | Description                                                     |
| ------------------ | :-------------------: | :-----: | --------------------------------------------------------------- |
| `command`          | `string`              | `task`  | the Taskwarrior executable to use                               |
| `tags`             | `map[string]string`   | see below | map of tag name to Taskwarrior filter; `count` is appended automatically |
| `context_command`  | `string`              | `""`    | arguments passed to `task` to retrieve the active context; omit to skip |

### Default `tags` value

```json
{
  "due": "+PENDING due.before:tomorrow",
  "scheduled": "+PENDING scheduled.before:tomorrow",
  "waiting": "+WAITING"
}
```

Only the tags present in `tags` result in a `task` command execution, so remove entries you do not need
to keep prompt rendering fast.

## Template ([info][templates])

:::note default template

```template
 {{ "\uf4a0" }} {{ range $k, $v := .Tags }}{{ $k }}:{{ $v }} {{ end }}
```

:::

### Properties

| Name       | Type              | Description                                                                      |
| ---------- | ----------------- | -------------------------------------------------------------------------------- |
| `.Tags`    | `map[string]int`  | task counts keyed by tag name with the first letter uppercased (e.g. `"Due"`)    |
| `.Context` | `string`          | the active Taskwarrior context; empty string when `context_command` is not set   |

### Examples

Access a specific tag directly:

```template
 {{ "\uf4a0" }} Due: {{ .Tags.Due }}
```

Display multiple specific tags:

```template
 {{ "\uf4a0" }} Due: {{ .Tags.Due }} | Waiting: {{ .Tags.Waiting }}
```

Include the active context:

```template
 {{ "\uf4a0" }} {{ .Context }} - Due: {{ .Tags.Due }} Scheduled: {{ .Tags.Scheduled }}
```
